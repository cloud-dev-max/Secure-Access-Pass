## 🛑 Issue #5: Enhanced Pool Closure

**Problem:** Staff could close pool without providing reason  
**Solution:** Prompt for mandatory reason, store in DB, display to residents

### **Implementation:**

#### **Scanner (app/scanner/page.tsx):**
```typescript
const togglePoolStatus = async () => {
  let reason = ''
  
  // V7.5: Prompt for reason when closing
  if (isPoolOpen) {
    reason = prompt('Please enter a reason for closing the pool:')
    if (reason === null) return // User cancelled
    if (!reason.trim()) {
      alert('Reason is required to close the pool')
      return
    }
  }
  
  // Save to database
  await fetch('/api/settings', {
    method: 'PATCH',
    body: JSON.stringify({
      is_maintenance_mode: !isPoolOpen,
      maintenance_reason: isPoolOpen ? reason : '',
    }),
  })
}
```

#### **Resident Portal (app/resident/page.tsx):**
```jsx
<div className={facilityStatus?.is_open 
  ? 'bg-green-600' 
  : 'bg-red-600'}>
  <h2>{facilityStatus?.is_open ? 'POOL OPEN' : 'POOL CLOSED'}</h2>
  
  {/* V7.5: Display closure reason */}
  {facilityStatus?.is_maintenance_mode && (
    <p>{facilityStatus.maintenance_reason}</p>
  )}
</div>
```

### **User Experience:**
1. Staff clicks "Close Pool" button on scanner
2. Prompted: "Please enter a reason for closing the pool:"
3. Staff enters: "Maintenance" or "Weather" or "Emergency"
4. Reason saved to `maintenance_reason` field
5. Residents see: **"POOL CLOSED - Maintenance"** in red banner

## 💡 Troubleshooting

### **Issue: Still seeing PGRST205 error**
**Solution:**
1. Verify table is named `visitor_passes` in Supabase
2. Check RLS policies allow admin client access
3. Confirm SUPABASE_SERVICE_ROLE_KEY is set correctly
4. Rebuild and redeploy: `npm run build && npx wrangler pages deploy dist`

### **Issue: Revenue tab still blank**
**Solution:**
1. Hard refresh browser (Ctrl+Shift+R)
2. Clear browser cache
3. Check browser console for API errors
4. Verify /api/revenue endpoint is accessible

### **Issue: Broadcasts still showing resident names**
**Solution:**
1. Check activity log in database
2. Verify `qr_code = 'SYSTEM_BROADCAST'`
3. Ensure `user_id` is null for broadcasts
4. Rebuild frontend: `npm run build`
