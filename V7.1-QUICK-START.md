# 🚀 V7.1 Quick Start Guide

## Download & Setup (5 minutes)

### 1. Download Project
**URL**: https://www.genspark.ai/api/files/s/8OXRb8wm
**File**: `webapp-v7.1-all-fixes-complete.tar.gz` (1.1 MB)

Extract the archive → creates `webapp` folder.

---

### 2. Install Dependencies
```bash
cd webapp
npm install
```
**Time**: ~2 minutes

---

### 3. Configure Supabase
**Get your credentials**:
1. Go to https://supabase.com/dashboard
2. Select your project → Settings → API
3. Copy:
   - **Project URL** (e.g., `https://xxxxx.supabase.co`)
   - **anon public** key
   - **service_role** key

**Update `.env.local`**:
```bash
NEXT_PUBLIC_SUPABASE_URL=https://YOUR-PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...your-anon-key...
SUPABASE_SERVICE_ROLE_KEY=eyJ...your-service-key...
NEXT_PUBLIC_DEFAULT_PROPERTY_ID=00000000-0000-0000-0000-000000000001
```

---

### 4. Run Migration (REQUIRED)
**Open Supabase SQL Editor**:
1. Go to https://supabase.com/dashboard
2. Your Project → SQL Editor → New Query
3. Copy & paste this SQL:

```sql
-- V7.1 Migration: Personal Limits & Visitor Passes
ALTER TABLE profiles 
ADD COLUMN IF NOT EXISTS personal_guest_limit INTEGER DEFAULT NULL;

ALTER TABLE properties 
ADD COLUMN IF NOT EXISTS max_visitor_passes INTEGER DEFAULT 10;

ALTER TABLE profiles 
ADD COLUMN IF NOT EXISTS last_scan_at TIMESTAMP WITH TIME ZONE DEFAULT NULL;

-- Performance indexes
CREATE INDEX IF NOT EXISTS idx_profiles_last_scan_at 
ON profiles(last_scan_at DESC);

CREATE INDEX IF NOT EXISTS idx_guest_passes_expires_at 
ON guest_passes(expires_at);

CREATE INDEX IF NOT EXISTS idx_profiles_current_location 
ON profiles(current_location) WHERE current_location = 'INSIDE';
```

4. Click **Run**
5. Verify: `SELECT * FROM properties LIMIT 1;` should show `max_visitor_passes` column

---

### 5. Start Development Server
```bash
npm run dev
```
Open http://localhost:3000

## Troubleshooting

**"Failed to load data"**:
- Check `.env.local` has correct Supabase keys
- Verify tables exist in Supabase Table Editor

**"Column does not exist: personal_guest_limit"**:
- Run the migration SQL again

**"Failed to send broadcast"**:
- See `BROADCAST-FIX.md` for dropdown fix

**Guest pass count wrong**:
- Clear browser cache
- Verify migration ran (check `last_scan_at` column exists)
