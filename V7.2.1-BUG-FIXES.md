## 🐛 Bug #1: Dashboard Runtime Crash - `loadingInsideResidents` Not Defined

### **Problem**
```
ReferenceError: loadingInsideResidents is not defined
Location: app/dashboard/page.tsx:1631
Impact: CRITICAL - Dashboard crashes on Occupancy tab load
```

### **Root Cause Analysis**

When Version 7.2 converted the "Who is Inside?" modal to a full-page Occupancy tab:
1. ✅ New Occupancy tab content was added (lines 1597-1725)
2. ✅ Tab navigation button was added
3. ❌ **Old modal code was NOT removed** (53 lines of dead code)
4. ❌ **State variables were partially removed**:
   - `showInsideModal` state remained but was unused
   - `loadingInsideResidents` state was **NEVER created**
   - `loadInsideResidents()` function still referenced the old modal
5. ❌ Occupancy tab referenced `loadingInsideResidents` which didn't exist

**Result:** Runtime crash when clicking "Refresh Occupancy List" button

### **The Fix**

**File:** `app/dashboard/page.tsx`

#### 1. Added Missing State Variable (Line 78)
```typescript
// BEFORE (V7.2)
const [showInsideModal, setShowInsideModal] = useState(false)
const [insideResidents, setInsideResidents] = useState<ProfileWithRules[]>([])

// AFTER (V7.2.1)
const [insideResidents, setInsideResidents] = useState<ProfileWithRules[]>([])
const [loadingInsideResidents, setLoadingInsideResidents] = useState(false)
```

**Changes:**
- ✅ Removed `showInsideModal` (unused after modal removal)
- ✅ Added `loadingInsideResidents` (required by Occupancy tab)

#### 2. Updated `loadInsideResidents` Function (Lines 318-327)
```typescript
// BEFORE (V7.2) - Opened modal
const loadInsideResidents = async () => {
  const inside = residents.filter(r => r.current_location === 'INSIDE')
  setInsideResidents(inside)
  setShowInsideModal(true) // ❌ Opened modal (removed in V7.2)
}

// AFTER (V7.2.1) - Uses loading state
const loadInsideResidents = async () => {
  setLoadingInsideResidents(true)
  try {
    const inside = residents.filter(r => r.current_location === 'INSIDE')
    setInsideResidents(inside)
  } finally {
    setLoadingInsideResidents(false)
  }
}
```

**Changes:**
- ✅ Added proper loading state management
- ✅ Removed modal trigger (`setShowInsideModal(true)`)
- ✅ Added try-finally for cleanup

#### 3. Removed Entire Old Modal (Lines 1727-1780)
```typescript
// BEFORE (V7.2) - 53 lines of dead code
{showInsideModal && (
  <div className="fixed inset-0 bg-black/50 ...">
    <div className="bg-white rounded-2xl ...">
      <h3>Who is Inside?</h3>
      {/* Modal content with Force Check Out buttons */}
    </div>
  </div>
)}

// AFTER (V7.2.1) - Clean comment
{/* V7.2: Leftover modal removed - use Occupancy tab instead */}
```

**Changes:**
- ✅ Removed 53 lines of unused modal JSX
- ✅ Replaced with explanatory comment
- ✅ Reduced bundle size by ~1.2 KB

#### 4. Updated Force Checkout Call (Line 344)
```typescript
// BEFORE
await loadData()
loadInsideResidents() // Synchronous call

// AFTER
await loadData()
await loadInsideResidents() // Async call with loading state
```

### **Result**
- ✅ Dashboard loads without crash
- ✅ Occupancy tab displays properly
- ✅ "Refresh Occupancy List" button shows spinner
- ✅ Force Exit works correctly
- ✅ No more runtime errors

## 🐛 Bug #2: Max Visitor Passes Setting Reverting to Default

### **Problem**
```
Issue: Manager changes "Maximum Visitor Passes" from 10 to 50
Expected: Value persists after save and page refresh
Actual: Value reverts back to 10 on page refresh
Impact: HIGH - Setting cannot be changed
```

### **Root Cause Analysis**

This was a **three-part problem** in the data pipeline:

#### 1. ❌ API GET Route (Read)
```typescript
// app/api/settings/route.ts Line 18
.select('..., max_guests_per_resident, is_maintenance_mode, ...')
//                                    ❌ max_visitor_passes MISSING
```
**Problem:** API didn't fetch `max_visitor_passes` from database

#### 2. ❌ API PATCH Route (Write)
```typescript
// app/api/settings/route.ts Line 67-76
const {
  property_name,
  operating_hours_start,
  // ...
  max_guests_per_resident,
  // ❌ max_visitor_passes NOT destructured
  is_maintenance_mode,
} = body
```
**Problem:** API didn't extract or save `max_visitor_passes` to database

#### 3. ❌ Settings UI (Display/Input)
```typescript
// app/dashboard/settings/page.tsx
// ❌ No maxVisitorPasses state variable
// ❌ No form field for max_visitor_passes
// ❌ Not included in save request payload
```
**Problem:** UI had no way to display or edit the setting

#### 4. ✅ Database Schema (Already Correct)
```sql
-- migrations/0007_v7.1_personal_limits.sql Line 24-25
ALTER TABLE properties
ADD COLUMN IF NOT EXISTS max_visitor_passes INTEGER DEFAULT 100;
```
**Note:** Database column existed and was correct! Only API/UI were broken.
