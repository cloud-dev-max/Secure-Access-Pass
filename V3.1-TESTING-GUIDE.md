### ✅ Test 1.1: Scanner Shows Resident Names (Bug Fix)

**Issue**: Scanner was showing "Unknown" instead of real names

**Steps**:
1. Navigate to `/scanner`

3. Scan any resident QR code
4. **Expected**: Should display actual resident name (e.g., "John Smith")
5. **Fail Condition**: Displays "Unknown" or null

**Manual Test (without camera)**:
```bash
# Test API directly
curl -X POST http://localhost:3000/api/check-access \
  -H "Content-Type: application/json" \
  -d '{
    "qr_code": "RES-1234567890",
    "scan_type": "ENTRY"
  }'

# Response should include:
{
  "can_access": true,
  "user_name": "John Smith",  // ✅ REAL NAME
  "user_id": "uuid-here"
}
```

**Pass Criteria**: ✅ Real name appears, not "Unknown"

### ✅ Test 3.2: Occupancy Counts by Property

**Requirement**: Occupancy should only count residents in same property

**Setup**:
```sql
-- Set 3 residents to INSIDE in property 1
UPDATE profiles
SET current_location = 'INSIDE'
WHERE property_id = '00000000-0000-0000-0000-000000000001'
LIMIT 3;

-- Set 2 residents to INSIDE in property 2
UPDATE profiles
SET current_location = 'INSIDE'
WHERE property_id = '11111111-1111-1111-1111-111111111111'
LIMIT 2;
```

**Test**:
1. Navigate to `/resident` (property 1)
2. Check occupancy display
3. **Expected**: Shows "3 / 50 Residents" (not 5)
4. **Fail Condition**: Shows combined count

**API Verification**:
```bash

{
  "current_occupancy": 3,  // ✅ Only property 1 residents
```

**Pass Criteria**: ✅ Occupancy counts are property-scoped

## Failure Protocols

### If Any Test Fails

1. **Note the failure** (screenshot + console logs)
2. **Check environment variables** (SERVICE_ROLE_KEY set?)
3. **Check database** (migrations applied?)
4. **Check browser console** (JavaScript errors?)
5. **Check network tab** (API errors?)
6. **Report issue** with:
   - Test case name
   - Expected vs actual behavior
   - Console logs
   - Network logs

### Critical Failures (Stop Testing)
- Scanner shows "Unknown" names
- Settings won't save
- Dashboard won't load
- APIs returning 500 errors

### Minor Failures (Continue Testing)
- Styling issues
- Text typos
- Non-critical UI glitches
