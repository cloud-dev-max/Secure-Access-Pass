## 🔧 Technical Details

### New API Endpoint

**`GET /api/logs`**
- **Purpose**: Fetch paginated activity logs
- **Authentication**: Admin client (uses `createAdminClient()`)
- **Query Parameters**:
  - `page`: Page number (default: 1)
  - `limit`: Logs per page (default: 50, max: 100)
- **Response Format**:
  ```typescript
  {
    logs: Log[],
    total: number,
    page: number,
    limit: number,
    totalPages: number
  }
  ```
- **Features**:
  - Joins with `profiles` table for user data
  - Ordered by `scanned_at` DESC
  - Filters by `property_id`
  - Includes total count for pagination

### Logs Page Features

**`/logs` Route**
- Client-side rendering (`'use client'`)
- Auto-loads on mount with default page 1
- Pagination state management
- Color-coded log types with icons
- Responsive design with Tailwind CSS
- Loading states with spinner
- Error handling
- Date formatting with `Intl.DateTimeFormat`

### Log Type Detection Logic

```typescript
const isStatusChange = log.qr_code === 'STATUS_CHANGE' || 
  (log.denial_reason && log.denial_reason.includes('Status changed from'))
  
const isSystemBroadcast = log.qr_code === 'SYSTEM_BROADCAST' || 
  (log.denial_reason && log.denial_reason.includes('BROADCAST'))
```
